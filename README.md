# buddhism in color

## Data

- Translations
- Root texts
- People
- Places
- Parallel texts
- Concepts
- Sentiment analysis
- Difficulty
- Blurbs
- Tags (manually created by going through each sutta; eg: #remote-lodgings, #fear)
- Word counts

Idea is that someone can get an impression of a sutta by skimming the tags, concepts and blurb. Tags and concepts will be shorter length than blurbs obviously.

When the sutta is being rendered, people / places / concepts etc. will need to be marked up programatically with event handlers attached.

### People

Check Michael's repo (suttamap)[https://github.com/michaelh-sc/suttamap/blob/main/data].

- `suttas_per_person_words_spoken.json`
